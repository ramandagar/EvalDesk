Agent eval tool build plan — what exists, the gap, what to build

What exists today — the honest truth

DeepEval, MLflow, Langfuse

Open source, powerful, well-maintained — but 100% engineer-only. You write Python/JSON test cases. No UI for domain experts.

code required

Confident AI, Maxim AI

No-code UI, domain experts CAN use them — but closed source, $500–$1000+/month. Not an option for startups or self-hosting.

paid + closed

The exact gap

Open source + self-hostable + no-code UI for domain experts. This combination does not exist. DeepEval is the closest but explicitly has "no UI, no collaboration, no production monitoring."

What you build — EvalDesk (working name)

A simple, self-hostable web app. A doctor, lawyer, teacher, or compliance officer opens the browser, types test questions, rates the AI answers, and sees a quality score over time. Zero code required from them.

Test case editor

Write Q&A pairs in plain English, no JSON, no code

One-click run

Hit any agent via URL — paste endpoint, add API key, done

Human rating UI

Good / Bad / Partial with comment. Fast keyboard shortcuts

Quality dashboard

Pass rate over time, worst-performing questions, regressions

Auto judge

Optional LLM-as-judge to score answers automatically, no human needed

Team sharing

Invite a doctor, lawyer, PM by email — no GitHub account needed

How to build it — step by step
1

Week 1–2: core backend

Next.js + TypeScript. SQLite for local, Postgres for cloud. Three tables: projects, test_cases, runs. One API route: POST /run — takes endpoint URL + test cases, fires requests, stores responses.

2

Week 3: rating UI

Show question + agent answer side by side. Three buttons: Pass / Fail / Partial. Comment field. Keyboard: 1/2/3 to rate fast. This is the core value — domain experts can fly through 50 test cases in 10 minutes.

3

Week 4: dashboard + Docker

Pass rate chart over time. Worst-performing test cases sorted by fail rate. Regression alerts when a new run scores lower than the last. Package everything as a single docker-compose.yml — one command to self-host.

4

Week 5: launch

Write a great README with a 30-second GIF of a non-engineer rating AI answers. Post on Hacker News "Show HN". Share in r/MachineLearning, r/LocalLLaMA, and 3 Discord servers (LangChain, Mastra, AI Engineer). Write one blog post: "Why your AI agent needs a doctor to test it, not a developer."
